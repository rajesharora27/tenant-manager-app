Authenticate with SSE API

Authenticate with your Cisco SSE API credentials to access tenant management features. Your credentials are stored securely in environment variables.

Authentication Details
API Base URL: {{ config.BASE_URL }}
Token URL: {{ config.TOKEN_URL }}
Username: {{ config.USERNAME|string|truncate(12, True, '...') }}
SSL Verification: {% if config.VERIFY_SSL == 'true' %} Enabled {% else %} Disabled {% endif %}
Grant Type: client_credentials
Authentication: Basic Auth (OAuth 2.0)
Configuration Required

Make sure your .env file contains:

  • BASE_URL - SSE API base URL
  • USERNAME - Your API client ID
  • PASSWORD - Your API client secret
  • TOKEN_URL - OAuth token endpoint
Authentication tokens are automatically managed and renewed as needed.
Troubleshooting

Possible causes:

  • Incorrect USERNAME or PASSWORD in .env file
  • API credentials have been revoked or expired
  • Wrong TOKEN_URL endpoint

Solutions:

  • Verify your API credentials in the .env file
  • Check with your administrator for valid credentials
  • Ensure the TOKEN_URL is correct for your environment

Possible causes:

  • Network connectivity issues
  • Firewall blocking the API endpoints
  • Incorrect BASE_URL or TOKEN_URL

Solutions:

  • Check your internet connection
  • Verify the URLs are accessible from your network
  • Contact your network administrator about firewall rules

Sample .env file:

BASE_URL=https://api.sse.cisco.com/
USERNAME=your-client-id-here
PASSWORD=your-client-secret-here
TOKEN_URL=https://api.sse.cisco.com/auth/v2/token
SECRET_KEY=your-flask-secret-key

Note: Replace the placeholder values with your actual SSE API credentials.
